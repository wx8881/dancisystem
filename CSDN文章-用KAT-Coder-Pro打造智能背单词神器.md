## 🎯 项目概述：这不是普通的背单词APP

### 项目名称
**智能英语词汇记忆系统 (Smart Vocabulary Memorization System)**

### 项目定位
这不是一个简单的背单词APP，而是一个**基于认知科学和机器学习算法的智能学习系统**。它能够：
- 🧠 **理解你的记忆规律**：通过算法分析你的学习数据，找出你的记忆模式
- 📈 **动态调整学习计划**：不是死板的"今天学20个"，而是根据你的状态智能推荐
- 🎯 **精准预测遗忘点**：在你还记得的时候提醒你复习，而不是等忘了再学

### 我解决了哪些痛点？

#### 痛点1：复习时机不科学
**传统方式**：今天学了20个单词，明天再学20个，之前的？看心情复习...  
**我的方案**：系统根据艾宾浩斯遗忘曲线，**精确计算每个单词的最佳复习时间**。比如"ability"这个单词，系统会在第1天、第3天、第7天、第15天、第30天自动提醒你复习，确保在遗忘临界点之前巩固记忆。

#### 痛点2：不知道自己的学习效果
**传统方式**：学了1000个单词，但不知道真正掌握的有多少...  
**我的方案**：**多维度数据统计**，包括：
- 学习量趋势图（看你的学习曲线）
- 准确率变化（看你的进步速度）
- 记忆强度分布（看哪些单词需要重点复习）
- 错误类型分析（看你在哪些方面容易出错）

#### 痛点3：错题本形同虚设
**传统方式**：错题本写满了，但从来不看，或者看了也不知道怎么改进...  
**我的方案**：**智能错题本**不仅记录错题，还：
- 分析错误类型（是理解错误还是记忆错误？）
- 追踪错误频率（哪些单词反复出错？）
- 对比正确答案（你的答案 vs 正确答案，一目了然）
- 自动生成针对性练习（专门针对你的薄弱点）

#### 痛点4：缺乏学习动力
**传统方式**：学了两天就坚持不下去了...  
**我的方案**：**游戏化学习机制**：
- 连续打卡天数（培养习惯）
- 学习成就系统（完成目标有成就感）
- 可视化进度条（看到自己的进步）
- 周报月报（定期总结，看到成长）

### 技术亮点（这才是最牛逼的！）

1. **自研记忆算法**：不是简单的"1天、3天、7天"固定间隔，而是根据每个用户的学习表现，**动态调整记忆强度（memory_strength）**，实现真正的个性化学习。

2. **企业级架构**：采用Oracle数据库 + FastAPI + Next.js的**全栈架构**，支持高并发、数据安全、可扩展。

3. **实时数据同步**：前端React 19 + 后端FastAPI，实现**毫秒级数据同步**，学习记录实时保存，不怕丢数据。

4. **智能推荐系统**：基于用户的学习历史和错误模式，**智能推荐需要重点复习的单词**，而不是随机抽取。

## ✨ 核心功能：每一个都是技术亮点

### 1. 🧠 智能学习模块：不只是"看单词"

这个模块看起来简单，但背后有**复杂的算法支撑**：

**渐进式学习状态机**：
```
陌生 → 学习中 → 认识/不认识 → 已掌握
  ↓       ↓         ↓           ↓
首次   标记状态   记录结果   加入复习计划
```

系统会为每个单词维护一个**学习状态机**，根据你的反馈自动流转。比如：
- 你标记"认识" → 系统记录到StudyLog，memory_strength设为0.6
- 你标记"不认识" → memory_strength设为0.2，系统会在1天后强制复习
- 你标记"学习中" → memory_strength设为0.4，系统会在3天后提醒复习

**实时数据同步**：每次操作都会**立即同步到Oracle数据库**，不会丢失任何学习记录。我用了FastAPI的异步特性，即使同时有100个用户在学习，系统也能流畅运行。

### 2. 📅 基于艾宾浩斯遗忘曲线的复习计划：这才是核心算法！

这是整个系统**最牛逼**的部分！我实现了一个**自适应的记忆算法**，不是简单的固定间隔，而是根据你的表现动态调整。

#### 算法原理

**传统方式**（市面上大多数APP）：
```
学完单词 → 1天后复习 → 3天后复习 → 7天后复习（固定）
```

**我的算法**（真正的智能）：
```python
# 伪代码展示核心逻辑
def calculate_next_review(memory_strength, repeat_count, is_correct):
    if is_correct:
        memory_strength += 0.1  # 答对了，记忆强度+0.1
        if memory_strength > 1.0:
            memory_strength = 1.0  # 上限1.0
    else:
        memory_strength -= 0.2  # 答错了，记忆强度-0.2
        if memory_strength < 0.1:
            memory_strength = 0.1  # 下限0.1
    
    # 根据记忆强度决定复习间隔
    if memory_strength < 0.3:
        next_review = current_time + 1 day  # 记忆很弱，1天后复习
    elif memory_strength < 0.6:
        next_review = current_time + 3 days  # 记忆一般，3天后复习
    else:
        next_review = current_time + 7 days  # 记忆较强，7天后复习
    
    return next_review, memory_strength
```

**实际效果**：
- 如果你连续3次答对某个单词 → memory_strength会从0.5涨到0.8 → 复习间隔从3天延长到7天
- 如果你答错了 → memory_strength会从0.6降到0.4 → 复习间隔从7天缩短到3天

**这才是真正的个性化学习！** 每个单词的复习计划都是**为你量身定制**的。

#### 技术实现细节

我在Oracle数据库中设计了`ReviewSchedule`表，包含：
- `memory_strength`：记忆强度（0.1-1.0）
- `repeat_count`：复习次数
- `review_date`：下次复习时间（精确到秒）

系统会**每天凌晨自动扫描**，找出需要今天复习的单词，推送给用户。这个定时任务我用了Python的`schedule`库实现。

**错误类型智能识别**：
- **含义理解错误**：你选了相近但不对的答案（比如把"abandon"理解成"ability"）
- **记忆错误**：你完全不记得这个单词
- **拼写错误**：你知道意思但拼错了（未来功能）
- **用法错误**：你知道意思但用错了（未来功能）

**错误频率追踪**：
系统会记录每个单词的**错误次数**，比如：
- "abandon"错了3次 → 系统会**优先推荐**这个单词给你复习
- "ability"错了1次 → 正常复习频率

**答案对比分析**：
```
你的答案：能力
正确答案：放弃
错误类型：含义理解错误
错误次数：3次
建议：重点复习，建议查看例句加深理解
```

**自动生成针对性练习**：
系统会根据你的错题，**自动生成专项练习**。比如你总是在"含义理解"上出错，系统会专门给你出这类题目。

# 这是KAT-Coder-Pro生成的算法，我稍微优化了一下
def calculate_next_review(memory_strength: float, is_correct: bool) -> tuple:
    """
    根据记忆强度和答题结果，计算下次复习时间
    """
    if is_correct:
        memory_strength = min(memory_strength + 0.1, 1.0)  # 答对了，记忆强度+0.1
    else:
        memory_strength = max(memory_strength - 0.2, 0.1)  # 答错了，记忆强度-0.2
    
    # 根据记忆强度决定复习间隔
    if memory_strength < 0.3:
        days = 1  # 记忆很弱，1天后复习
    elif memory_strength < 0.6:
        days = 3  # 记忆一般，3天后复习
    else:

## 🚀 快速开始

### 环境要求
- Python 3.8+
- Node.js 18+
- Oracle Database 12c+

### 安装步骤

1. **克隆项目**
```bash
git clone [项目地址]
cd dancisystem
```

2. **安装后端依赖**
```bash
cd backend
pip install -r requirements.txt
```

3. **配置数据库**
```bash
# 执行数据库脚本
sqlplus system/111111@localhost:1521/FREE @database-schema.sql
sqlplus system/111111@localhost:1521/FREE @init_data.sql
```

4. **启动后端服务**
```bash
cd backend
uvicorn main:app --reload --port 8000
```

5. **安装前端依赖**
```bash
cd front
npm install --legacy-peer-deps
```

6. **启动前端服务**
```bash
cd front
npm run dev
```

7. **访问系统**
- 前端：http://localhost:3000
- 后端API：http://localhost:8000
- API文档：http://localhost:8000/docs
